# EDGE TRADER
**Техническая документация · v2.1**
Совместно с Edge Protocol & Debunk and Bet API

## 1. Технологический стек
Бот работает на том же VPS, что и основной бэкенд Edge Protocol, и подключается к существующему инстансу PostgreSQL через отдельную схему/базу для изоляции данных.

| Компонент | Технология | Примечание |
| :--- | :--- | :--- |
| **Runtime** | Node.js 20+ (TypeScript) | LTS, совместим с текущим VPS. |
| **HTTP фреймворк** | Fastify | Быстрее Express, нативный async/await. |
| **PostgreSQL клиент** | `pg` (node-postgres) + Pool | Стандарт для TypeScript; подключается к существующей БД. |
| **Process manager** | `pm2` | Уже установлен на VPS. |
| **HTTP клиент** | `axios` / `fetch` | Для вызовов D&B API и CLOB API. |

**Подключение к БД:**
```typescript
// src/db.ts
import { Pool } from 'pg';

export const pool = new Pool({
  connectionString: process.env.BOT_DATABASE_URL,
  max: 10,
});
```

## 2. Структура базы данных

*В этой версии убраны ограничения `NOT NULL` для полей `debunk_verdict` и `edge`, чтобы позволить первичному `INSERT` создавать пустые заявки до прихода webhook. Добавлена строгая проверка диапазона для `confidence`.*

```sql
CREATE TABLE virtual_trades (
    id                UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id           VARCHAR(255)  UNIQUE NOT NULL,
    market_url        TEXT          NOT NULL,
    category          VARCHAR(100),
    side              VARCHAR(10)   CHECK (side IN ('YES', 'NO')),

    -- Сигнал от D&B API (заполняется после webhook)
    debunk_verdict    DECIMAL(6,4),   
    edge              DECIMAL(6,4),   
    kelly_fraction    DECIMAL(6,4),
    confidence        VARCHAR(20)   CHECK (confidence IN ('normal', 'low')),
    prior_n           INTEGER,
    confidence_mult   DECIMAL(4,3)  CHECK (confidence_mult BETWEEN 0.200 AND 1.000),

    -- Исполнение (заполняется после webhook)
    entry_price       DECIMAL(6,4),
    entry_volume_usdc DECIMAL(10,2),
    target_price      DECIMAL(6,4),

    -- Статус и результат
    status            VARCHAR(20)   DEFAULT 'PENDING'
                      CHECK (status IN ('PENDING','OPEN','CLOSED_EDGE','CLOSED_TIMEOUT','ERROR')),
    error_reason      TEXT,
    exit_price        DECIMAL(6,4),
    pnl_usdc          DECIMAL(10,2),

    entry_time        TIMESTAMPTZ,
    exit_time         TIMESTAMPTZ,
    created_at        TIMESTAMPTZ   DEFAULT NOW()
);

CREATE INDEX idx_status     ON virtual_trades(status);
CREATE INDEX idx_task_id    ON virtual_trades(task_id);
CREATE INDEX idx_entry_time ON virtual_trades(entry_time DESC);
```

## 3. Описание модулей

### Модуль 1 — Поисковый цикл (Gamma API)
* **Триггер:** Cron (каждые N минут, настраивается через `.env`).
* **Логика выполнения:**
    * Запрашивает открытые рынки через Gamma API.
    * Фильтрует по категории и ликвидности.
    * Отправляет URL подходящих рынков в D&B API (`/api/v1/analyze-bot`).
    * Создаёт запись в `virtual_trades` со статусом `PENDING`.

### Модуль 2 — Аналитический мост (D&B Bridge & Webhook)
**Отправка задачи в D&B API:**
```typescript
const response = await axios.post(
  `${process.env.DEBUNK_API_URL}/api/v1/analyze-bot`,
  { url: marketUrl, webhook_url: process.env.BOT_WEBHOOK_URL },
  { headers: { 'X-API-Key': process.env.DEBUNK_API_KEY } }
);
const taskId = response.data.task_id; // task_id из ответа D&B

await pool.query(
  `INSERT INTO virtual_trades (task_id, market_url, status) VALUES ($1, $2, 'PENDING')`,
  [taskId, marketUrl]
);
```
> **ВАЖНО:** `task_id` генерируется на стороне D&B API и возвращается в теле ответа `POST /analyze-bot`. Именно это значение сохраняется в БД и используется для сопоставления с входящим webhook.

**Приём webhook (`POST /webhook/signal`):**
```typescript
fastify.post('/webhook/signal', async (request, reply) => {
  // Верификация подлинности запроса
  const expectedKey = process.env.DEBUNK_API_KEY;
  if (request.headers['x-api-key'] !== expectedKey) {
    return reply.status(401).send({ error: 'Unauthorized' });
  }

  const signal = request.body as TradeSignal;

  if (signal.recommended_bet !== 'BET') {
    await pool.query(
      `UPDATE virtual_trades SET status = 'CLOSED_EDGE', error_reason = 'SKIP signal' WHERE task_id = $1`,
      [signal.task_id]
    );
    return reply.send({ ok: true });
  }

  await handleBetSignal(signal);
  return reply.send({ ok: true });
});
```

### Модуль 3 — Симулятор исполнения (Virtual Execution)
* **Триггер:** Вызов из webhook handler при `recommended_bet === 'BET'`.
* **Логика выполнения:**
    * Проходит по стакану CLOB API.
    * Вычисляет `entry_price` с учётом проскальзывания.
    * Устанавливает `target_price = entryPrice * TARGET_PROFIT_MULT` (из `.env`, по умолчанию 1.05).
    * Обновляет статус на `OPEN`, записывает все поля из сигнала D&B.

```typescript
async function handleBetSignal(signal: TradeSignal) {
  try {
    const { entryPrice, volume } = await simulateOrderBookEntry(
      signal.trade_direction, signal.kelly_fraction
    );

    const targetMult = parseFloat(process.env.TARGET_PROFIT_MULT ?? '1.05');

    await pool.query(
      `UPDATE virtual_trades
       SET status = 'OPEN', side = $1, entry_price = $2, entry_volume_usdc = $3,
           target_price = $4, entry_time = NOW(), debunk_verdict = $5, edge = $6,
           kelly_fraction = $7, confidence = $8, prior_n = $9, confidence_mult = $10
       WHERE task_id = $11`,
      [signal.trade_direction, entryPrice, volume, entryPrice * targetMult,
       signal.ai_prob_yes, signal.edge, signal.kelly_fraction, signal.confidence,
       signal.prior_n, signal.confidence_mult, signal.task_id]
    );
  } catch (err) {
    await pool.query(
      `UPDATE virtual_trades SET status = 'ERROR', error_reason = $1 WHERE task_id = $2`,
      [String(err), signal.task_id]
    );
  }
}
```

## 4. Стейт-машина бота

**Статусы виртуальных сделок:**
| Статус | Описание |
| :--- | :--- |
| `PENDING` | Задача отправлена в D&B API, ожидается webhook. |
| `OPEN` | Симуляция входа выполнена, позиция активна. |
| `CLOSED_EDGE` | Позиция закрыта: достигнут `target_price` или SKIP-сигнал. |
| `CLOSED_TIMEOUT` | Рынок истёк без достижения цели. |
| `ERROR` | Ошибка API/стакана, причина в `error_reason`. |

**Переходы статусов:**
* `PENDING` ──(SKIP webhook)─────────────────────► `CLOSED_EDGE`
* `PENDING` ──(BET webhook + успешный вход)──────► `OPEN`
* `PENDING` ──(ошибка стакана/API)───────────────► `ERROR`
* `PENDING` ──(webhook не пришёл > 1 часа)───────► `ERROR`
* `OPEN` ────(target достигнут)──────────────────► `CLOSED_EDGE`
* `OPEN` ────(таймаут рынка)─────────────────────► `CLOSED_TIMEOUT`
* `OPEN` ────(ошибка проверки)───────────────────► `ERROR`

**Рабочие процессы и Cron-очистка:**

```typescript
import cron from 'node-cron';

// Очистка зависших PENDING-записей каждые 30 минут
cron.schedule('*/30 * * * *', async () => {
  try {
    const result = await pool.query(`
      UPDATE virtual_trades
      SET status = 'ERROR', error_reason = 'Webhook timeout: D&B не ответил в течение 1 часа'
      WHERE status = 'PENDING' AND created_at < NOW() - INTERVAL '1 hour'
      RETURNING id;
    `);
    if (result.rowCount > 0) {
      console.log(`[Cron] Переведено в ERROR по таймауту webhook: ${result.rowCount} записей.`);
    }
  } catch (err) {
    console.error('[Cron] Ошибка очистки PENDING задач:', err);
  }
});
```

| Процесс | Тип | Частота | Задача |
| :--- | :--- | :--- | :--- |
| **Поисковый цикл** | Cron | Каждые N минут | Gamma API → фильтр → D&B API → `PENDING`. |
| **Webhook listener** | Event-driven | 24/7 (по событию) | `TradeSignal` → верификация → симуляция → `OPEN`. |
| **Торговый цикл** | Cron | Каждые 15–30 сек | `SELECT OPEN` → CLOB проверка → `CLOSED` / `ERROR`. |
| **Cleanup (>1ч)** | Cron | Каждые 30 мин | `PENDING` + таймаут → `ERROR` (новый процесс). |

```env
DEBUNK_API_URL=http://localhost:8000
DEBUNK_API_KEY=your_secret_api_key

# ─── Webhook ────────────────────────────────────────────────────
# Используйте localhost, если бот на том же VPS, иначе — внешний домен
BOT_WEBHOOK_URL=http://localhost:3000/webhook/signal
BOT_PORT=3000

# ─── Торговые параметры ─────────────────────────────────────────
TARGET_PROFIT_MULT=1.05   # +5% от entry_price (НОВОЕ: вынесено из кода)

# ─── Google Sheets (репортинг) ──────────────────────────────────
GOOGLE_SHEET_ID=your_sheet_id
GOOGLE_SERVICE_ACCOUNT_KEY=path_to_json_or_base64_string
```

## 6. Модуль репортинга (Google Sheets)
Переменные `GOOGLE_SHEET_ID` и `GOOGLE_SERVICE_ACCOUNT_KEY` используются модулем периодического репортинга.

| Событие | Записываемые поля |
| :--- | :--- |
| **Закрытие сделки** (`CLOSED_EDGE` / `CLOSED_TIMEOUT`) | `task_id, market_url, side, entry_price, exit_price, pnl_usdc, entry_time, exit_time`. |
| **Ошибка** (`ERROR`) | `task_id, market_url, error_reason, created_at`. |
| **Ежедневный дайджест** (по cron) | Суммарный PnL за день, количество сделок по статусам. |
